## Known Bug

**Netlify Identity invite email redirects to site root `/` instead of `/admin/`**

When a user clicks the invite/password-reset email link, they land on the homepage, not the admin panel. The redirect script in `index.html` only catches the `login` event from users already on the site.

### Fix needed

Add token detection to `index.html` that catches `invite_token` or `confirmation_token` in the URL hash and redirects to `/admin/`:

```javascript
if (window.location.hash && (window.location.hash.includes('invite_token') || window.location.hash.includes('confirmation_token'))) {
  window.location.href = '/admin/' + window.location.hash;
}
```

## Architecture

```
GitHub repo (main branch)
├── admin/index.html          ← Decap CMS (folder collections, Netlify Identity widget)
├── index.html                ← Main site + Netlify Identity redirect
├── data/gear/*.json          ← 65 individual gear items (CMS edits these)
├── data/packages/*.json      ← 5 individual packages (CMS edits these)
├── data/gear.json            ← Auto-generated by build.js (frontend reads this)
├── data/packages.json        ← Auto-generated by build.js (frontend reads this)
├── assets/*.webp             ← 67 product images + hero
├── build.js                  ← Merges individual JSONs → combined files
├── config.yml                ← Decap CMS config for local decap-server
├── netlify.toml              ← Build: node build.js, Publish: .
└── .gitignore

Client workflow:
  Edit in CMS → Decap commits to data/gear/*.json → Push to GitHub
  → Netlify auto-deploys → runs build.js → regenerates combined JSONs → site updated
```

## Key Files & Their Roles

| File | Role |
|------|------|
| `admin/index.html` | CMS entry point. `CMS_MANUAL_INIT`, `initCMS()`, Netlify Identity widget, folder collections config |
| `index.html` | Main website. Fetches `data/gear.json` + `data/packages.json`. Has Netlify Identity redirect script |
| `build.js` | Reads `data/gear/*.json` → writes `data/gear.json`. Same for packages. Runs on every Netlify deploy |
| `config.yml` | Local dev config for `decap-server`. Mirrors admin inline config |
| `netlify.toml` | `command = "node build.js"`, `publish = "."` |
| `data/gear/*.json` | 65 individual gear items. These are what the CMS reads/writes |
| `data/packages/*.json` | 5 individual packages. What the CMS reads/writes |
| `data/gear.json` | Combined file generated by `build.js`. What `index.html` fetches |
| `data/packages.json` | Combined file generated by `build.js`. What `index.html` fetches |
